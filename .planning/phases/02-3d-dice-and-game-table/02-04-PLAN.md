  <action>
Implement the full selection UX described in CONTEXT.md: tap to group, slide animation, dimming, shake/reject.

1. **Update DiceGroup.tsx -- Determine scoring dice after settle**:
   - After `setSettled(values)` is called, immediately compute scoring dice: `const scoringIndices = findScoringDice(values as Roll)`
   - Call `useDiceStore.getState().setCanScore(scoringIndices)`
   - Store each die's "resting position" (where it landed after physics) in a ref array: `restPositions.current[i] = body.translation()` -- this is the position the die returns to when deselected

2. **Define selected area layout**:
   - Selected dice slide to the LEFT side of the table (matching concept image 3: scored dice grouped on the left)
   - Selected area: x from -3.5 to -1.5, z stacked vertically
   - Calculate target positions for selected dice: arrange in a column on the left, spaced ~1.2 units apart in z
   - `getSelectedPosition(slotIndex: number)`: returns `[-2.5, 0.5, -2 + slotIndex * 1.2]` (adjust to fit within felt)
   - Track which "slot" each selected die occupies to avoid overlaps

3. **Update Die.tsx -- Add tap handling and animations**:
   - Switch RigidBody to `type="kinematicPosition"` after settle (so we can control position for slide animation). During rolling, it stays `type="dynamic"`.
   - **Tap handler** (`onPointerDown` on the die mesh):
     - If `phase !== 'settled'`, ignore taps
     - If die `canScore === true` and `selected === false`: call `selectDie(index)` -- die slides to selected area
     - If die `selected === true`: call `deselectDie(index)` -- die slides back to rest position
     - If die `canScore === false`: trigger shake animation (non-scoring die rejected)
   - **Slide animation** in `useFrame`:
     - Track a `targetPosition` ref for each die (either selected slot or rest position)
     - Use `THREE.MathUtils.lerp` with factor 0.12 (~0.3s ease at 60fps) to interpolate current position toward target
     - When selected changes, update targetPosition to the new target
     - For kinematic bodies, use `body.setNextKinematicTranslation({ x, y, z })` to move the die
     - Slide path: move in a slight arc (raise y by 0.3 during slide mid-point) to avoid clipping through other dice on the table
   - **Dimming** for unselected dice:
     - When ANY die is selected (store has at least one selected), dim all unselected dice
     - Dim by reducing material opacity: `material.opacity = selected ? 1.0 : 0.5` with `material.transparent = true`
     - Or reduce emissive/color intensity: multiply color by 0.5 for unselected
     - The simpler approach: adjust material color in useFrame based on selected state. Use `meshRef.current.material.color.lerp(targetColor, 0.1)` for smooth transition
     - Selected dice: full white `#f5f5f0`, Unselected dice: dimmed `#888880`
   - **Shake animation** for non-scoring dice:
     - On tap of non-scoring die, set a `shakeStartTime` ref to current time
     - In useFrame, for 300ms after shakeStartTime, oscillate the die's x position: `restPosition.x + Math.sin(elapsed * 30) * 0.1 * (1 - elapsed/0.3)` (fast shake that decays)
     - After 300ms, snap back to rest position

4. **Update diceStore.ts** -- Track selection order for slot assignment:
   - Add `selectionOrder: number[]` to store (list of die indices in order they were selected)
   - `selectDie`: pushes index to selectionOrder
   - `deselectDie`: removes index from selectionOrder, recalculates slots for remaining selected dice
   - This ensures slot positions are packed (no gaps when deselecting a middle die)

5. **Pointer events setup**:
   - Ensure Canvas in GameScene.tsx has `events` configured for touch devices (R3F handles this by default)
   - Use `onPointerDown` (not onClick) for faster mobile response
   - Set `cursor: pointer` style on the Canvas when hovering a scoring die (optional -- improves desktop UX)
